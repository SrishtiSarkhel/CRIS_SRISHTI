About Smms

    The smmss website is accessible through any major web browser viz. Google Chrome, Mozilla Firefox, Internet Explorer, Apple Safari etc.Bandwidth: Internet bandwidth minimum – 50 mbps through the Url: https://smms.rcil.gov.in/


Logging In

Login with the user ID (Login helps the user to quickly and easily manage user authentication and sign in to the user website).

This is the initial page that is displayed upon accessing the Smms website. Users can utilise the provided form on this page to log in to the website by entering their assigned user ID and password, provided by the CRIS admin. When the system configurations and requirements are met then the user can have access to the application by entering the provided URL in the compatible browsers. The user needs to enter a valid ‘Username’ ‘Password’ and correct ‘Captcha’ to gain access into the application.


Home Page (Dashboard)

When users logs in to the ‘SMMS’ application, user will land on the main page of an application based of roles and responsibility assigned to user. This page gives the broad information for Unhealthy Assets, Maintenance Overdue, Failure incidences and Codal life Outlived counts against the total counts of the assets Division wise. When the division labels are clicked it will land to the next page where more details of the Assets month-wise is displayed in graph formats. The different Menu on Web will change as per the rights provided to the user based on the designation. Accordingly Read only, Read-Write rights are given as per Rights Matrix. It will give a message as “You are not Authorized” for such menu’s which is not relevant to the User. Details are described in the User management Document. However Dashboard is visible to all the users.



To enter the main page of dashboard, user needs to successfully login to SMMS application. Then user shall be able to see the main page of dashboard. It contains the details of Zone, Division, and Section on the top side of the header. User can use this option to filter out the specific assets and get details about it.



Unhealthy Assets

Once the user selects the Zone, Division and section, they need to click on search button. Once the search button is clicked, the dashboard will check and highlight all the Unhealthy assets under all assets section. The total of 6 months of data will be shown onto the dashboard. Based on the measured parameters while performing maintenance, the system will derive the health of the assets. The blue colour signifies Total planned schedules, while the green colour signifies total performed assets and red colour suggest unhealthy assets. The total performed assets and unhealthy assets is always sum of total planned schedules. To summarise good assets are highlighted in green colour and red colour means that assets are critical and need attention. If user clicks on the red colour bar, than the data of respective month will be highlighted on the screen. The bar chart shows all the assets with the specific number of unhealthy assets.







For details list of Assets, above Button shall be clicked. It will give full details of the list of Assets as per the Tab selected above. Further drill down to Asset level details can be done by clicking the Gear number. Another window should be visible where one can see all the details of the past Maintenance and reports whichever is applicable.



The list can be exported by clicking the below shown Tab.



Maintenance Overdue

On the right side of unhealthy assets, user can find the option of Maintenance Overdue. User can find the details of all the assets which have overdue maintenance. All division’s overdue counts will be displayed above the bar. When user clicks on any of the bar, total last 6 months of overdue will be displayed and asset wise overdue counts will be displayed in bar chart. Further drill down till asset level should be done as explained above.



Failure Incidences

Failure assets will be shown of last 2 years i.e. last year and the current year. When the user lands on the main dashboard, they need to select location, division and section. After that besides maintenance overdue, one can find the option of Failure assets. All the assets that failed in last 2 years counts will be displayed on the screen. As shown in the below figure, click on any dot and respective month’s division wise failure counts will be displayed. In the below bar chat, asset wise overdue counts will be displayed. Further drill down till asset level should be done as explained above.



Codal life Outlived

Codal life expiry will be displayed in the gap of 3-6-9-12 months. While entering dashboard and after entering details of location, division and section. The assets whose codal life is getting expired in last 3-6-9-12 months will be displayed along with the health of the assets. In the figure, one can see the health in the first column and in the remaining column is the months. The health is displayed in the form of Codal life to be expire, good health, warning and critical.



GIS

TTo enter the main page of GIS, user needs to successfully login to SMMS application. Then user shall the main page of dashboard. GIS is the second option from the right hand side, as shown in the below figure.



GIS- Main Page

As user can see, the GIS main page consists of options to search the assets based on asset code. at left top corner of the page. At left bottom side of the page, user can see the longitude and latitude. Application shows latitude and longitude value of cursor position, it will change as cursor will move on a map. Where on the right side Zoom in, Zoom out, Layer and Filters functionality buttons are given. Please refer below figure


    Search Assets

    On the left top side of the page, user will enter the asset code of the asset which need to search and press search icon. Application will locate exact geo location of searched asset on a map is available.



    Basemap and vector layer

    On the right side of the page, user can find the option of basemap. By using this option user can change the background of the map, Google map, satellite map Open street map (OSM) options are available. Below the basemap option, user can find vector layers such as all assets, Railway Station, Railway Track, Zone and Failure Assets. User can display toggle these vector layers by enabling/disabling the tick mark.


    Filter Layers

    On the right side of the page and below the layer button, user can find the option to filter the features from specific layers like Assets, Railway station etc. there are two ways to search/filter the features from selected layers. 1. Search by Attributes and 2. Search by location.

    1. Search By Attributes: User can set the parameter, Operator and value. He can set multiple parameters also. He can use this option in combination with Search by Location option as well.
    2. Search by location: if user wants to search which and how much features are available in particular area. He will mark Rectangle, circle or polygon on a map after selecting feature layer. By clicking on search button system will filter the features which are intersect with marked geometry on a map. This option can be use in combination with search by attribute.


      3. General Info


    Once user clicks on i icon, new dialog box pops up which consists five options.

  • General Information: It shows all the information like asset code, Latitude and Longitude, gear number, modal, make, codal life, and more. Also, user will able to see image of the asset in the general info. Section.

  • Components: It gives details on type of lamps, Pilot aspects and ON aspects.

  • Documents: In this tab, user can easily upload the documents in form of PNG, JPEG, and PDF etc.

  • Failure: This tab shows the gear which are failed along with date of failure, Cause of failure, Repaired Status and its location

  • Replacement: It shows details on when the parts needs to be replaced with its replacement dates.
